Development Plan

1. Team & Project Overview

Team Name: AAIR FRYERS

Members: Asadel Ali, Roshan Azeemi, Ivy Zhang, Krishdeep Sawhney, Aneek

Client: Niko Corr

Application: One website with multiple pages for McMaster Jazz Club (Mac Jazz Club Organiser)

2. Purpose

Creating a convenient and accessible website for the members of the Mac Jazz Club. That allows the admin to track attendance, include a history (gallery page), and add a fun, competitive element with the leaderboard. The website will accept an Excel sheet and load the data into the database, allowing Niko to conveniently enter and modify data. We chose to build this product to make the lives of the Jazz Club admins easier, as they are able to see more data and enter a lot more data quickly. While also creating a webpage for the club that gives them more recognition.

3. Databases

Login / User Database

The client asked if it would be possible to have an admin login and a general user login so that he, as an admin, can view and edit anonymous information as per his discretion.

Anonymous Attendance (Instruments)

The client asked that the members' attendance be stored so he can see who plans to show up and how consistently people have shown up in the past.

Music Database (Gallery)

The gallery page is planned to include a mouseover feature that plays music while hovering over pictures. So this database will include possible music.

Leaderboard Database

This uses the attendance database to see which group of people has attended the most. (Based on percentage average, and not just the number of people).

Excel Data Database

This is for the client to drop in an Excel and for that data to be formatted into a database, such that the data gets added onto the webpage.

Events

Keep track of which events/practices group members have to attend.

4. Functions Performed by the App

The app will allow the user to use all four of the MySQL functions (INSERT, UPDATE, and DELETE), as they will need to be able to insert the data from the Excel page. They will also need to be able to add/update the data incase there are errors or people are unable to attend the event that they planned to initially attend. The user will also require the SQL function delete in case they need to delete data that is now irrelevant / not true (e.g. someone initially planned not to attend but ended up attending the practice/event).

8. Client Feedback

Attendance Privacy

Niko specified that "the attendance feature should not reveal people's names and only their instruments" (names should be hidden). We are going to implement this change by allowing members of the Mac Eng Jazz Club to log in and record their own attendance in the web app, then reveal the average number of instruments present for each section.

Database & Events

Niko said he "wants a database for the jazz club that checks the attendance for instruments/players and shows events/timeline". We are going to implement this feature using JS / CSS / PHP / MySQL.
